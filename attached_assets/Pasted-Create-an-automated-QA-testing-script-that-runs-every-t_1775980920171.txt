Create an automated QA testing script that runs every time it's called and reports bugs found.

Create a file: /scripts/qa-test.js

This script must:

## What to test automatically

1. API ENDPOINTS - Call each endpoint and verify it responds correctly:
   - GET /api/meals → must return array of 21 meal objects
   - GET /api/workouts → must return array of workout objects
   - GET /api/exercises/gif?name=squat → must return { gifUrl: string or null }
   - GET /api/flex-days → must return array
   - POST /api/flex-days → must accept { date } and return success
   - GET /api/progress → must return progress data
   - POST /api/stripe/create-checkout-session → must return { url: string }

2. DATABASE TABLES - Verify all required tables exist in both Supabase and local PostgreSQL:
   Supabase: profiles, food_preferences, meal_plans, workout_plans, progress_logs, calendar_events
   Local PostgreSQL: stripe_users, flex_days, workout_history, weekly_checkins

3. ENVIRONMENT VARIABLES - Verify all required env vars are set:
   - ANTHROPIC_API_KEY
   - SUPABASE_URL
   - SUPABASE_ANON_KEY
   - STRIPE_SECRET_KEY
   - STRIPE_PRICE_ID
   - RAPIDAPI_KEY

4. AI GENERATION - Test that Claude generates valid meal and workout plans:
   - Call /api/meals with a test profile
   - Verify response has 21 meals
   - Verify all meals have: day_name, meal_type, meal_name, ingredients, calories_approx
   - Verify meals are in Spanish
   - Call /api/workouts with a test profile
   - Verify response has correct number of workouts
   - Verify workouts are in Spanish

5. DATA INTEGRITY - Check for common data issues:
   - Meal plans with missing days
   - Workout plans with English exercise names
   - Progress logs with invalid dates
   - Profiles with missing required fields

6. FRONTEND ROUTES - Verify all pages exist and return 200:
   - / (landing page)
   - /dashboard
   - /meals
   - /workouts
   - /calendar
   - /progress
   - /shopping
   - /billing
   - /pricing

## Output format
At the end, print a clear report:

===== GOALIQ QA REPORT =====
Date: [timestamp]

✅ PASSING (X tests)
- API /api/meals: OK
- Database table profiles: OK
...

❌ FAILING (X tests)
- API /api/workouts: ERROR - returned 500
- Env var RAPIDAPI_KEY: MISSING
- Meal plan: Only 18 meals generated (expected 21)
...

⚠️ WARNINGS (X tests)  
- Workout plan has English exercise names
- Profile missing target_weight_kg field
...

===== SUMMARY =====
X passing, X failing, X warnings
Status: [HEALTHY / NEEDS ATTENTION / CRITICAL]

## How to run
Add to package.json scripts:
"qa": "node scripts/qa-test.js"

So it can be run with: npm run qa

In the sidebar bottom, add a small "🔍 QA Check" button that:

- On click: calls a new endpoint GET /api/qa that runs the tests server-side
