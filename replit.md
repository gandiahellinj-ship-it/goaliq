# GoalIQ Workspace

## Overview

pnpm workspace monorepo using TypeScript. GoalIQ (formerly NutriCoach) is a mobile-first PWA for AI-powered nutrition and training coaching.

- Name: **GoalIQ**
- Theme: Dark (`#0A0A0A` bg, `#1A1A1A` cards, `#2A2A2A` borders, `#AAFF45` lime accent, white text)
- Font: Barlow Condensed (headings, `font-display`, uppercase bold italic) + Inter (body)
- Logo: `<span>` — white "Goal" + lime "IQ"
- Stripe: €19.99/month, 3-day trial, price `price_1TFYJVAC9aQrlGDtdvlFPtjX`, portal `bpc_1TFYjxAC9aQrlGDtKsqblfb6`
- Supabase project: `bftggzsbovbjulbzyldj`

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **API framework**: Express 5
- **Database**: Supabase (PostgreSQL) via `@supabase/supabase-js` — user data accessed with per-request user JWTs (RLS enforced)
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec)
- **Build**: esbuild (CJS bundle)
- **Frontend**: React + Vite, Wouter, TanStack Query, Tailwind, Recharts
- **Auth**: Supabase Auth — email/password via `@supabase/supabase-js`; backend validates Bearer JWTs via `supabase.auth.getUser(token)`
- **AI**: Replit AI Integrations (OpenAI) for meal and workout plan generation

## Features

### Authentication
- Replit Auth (OIDC with PKCE)
- Sessions stored in PostgreSQL (sessions table)
- Browser: `useAuth()` hook from `@workspace/replit-auth-web`
- Login: redirect to `/api/login?returnTo=<BASE_URL>`
- Logout: redirect to `/api/logout`

### Data Layer
- **Frontend-first**: All pages (Meals, Workouts, Calendar, Progress, Dashboard) query Supabase directly
- `src/lib/supabase-queries.ts` — React Query hooks: `useMealPlan`, `useWorkoutPlan`, `useProgressLogs`, `useProgressStats`, `useProfile`, `useLogWeight`, `useToggleWorkoutComplete`
- `src/lib/onboarding-service.ts` — generates 7-day meal plans (21 rows) + workout plans on signup

### Onboarding
- Multi-step form: personal info → diet preferences → training preferences
- Saves to `profiles` + `food_preferences` tables; auto-generates meal/workout plans for current week
- Required before accessing meal/workout plans

### AI-Generated Plans
- Weekly meal plan generated by GPT-5-mini based on user profile
- Ingredient replacement via AI (same category, respects diet + allergies)
- Weekly workout plan generated by GPT-5-mini based on user profile
- Plans stored in PostgreSQL

### Calendar
- Monthly/weekly calendar view
- Workout days auto-populated when generating workout plan
- Toggle workout completion (affects adherence %)

### Progress
- Weight logging with history
- Weekly adherence tracking
- Line chart of weight over time

### Stripe Subscriptions
- 3 tiers: Basic ($4.99/mo), Pro ($9.99/mo), Premium ($19.99/mo)
- Stripe integration via Replit Connectors (credentials fetched dynamically via `REPLIT_CONNECTORS_HOSTNAME`)
- `stripe-replit-sync` syncs Stripe events to local PostgreSQL `stripe.*` schema via webhooks
- Webhook registered at `/api/stripe/webhook` (raw body handler BEFORE `express.json()`)
- Stripe products seeded via `scripts/src/seed-products.ts`
- `public.stripe_users` table tracks Supabase user ID ↔ Stripe customer ID mapping

#### Feature Gating
- **Basic**: today's meal only, today's workout only, weight logging
- **Pro**: full 7-day meal plan, ingredient swapping, progress chart, full calendar
- **Premium**: smart coaching insights card, weight trend analytics
- `UpgradeBanner` component (`src/components/UpgradeBanner.tsx`) shows locked-feature UI
- `useSubscription()` hook (`src/lib/subscription.ts`) fetches tier from `/api/subscription`

### Internationalisation (i18n)
- Full EN/ES support across every page and component
- `src/lib/language.tsx`: ~350+ translation keys, EN + ES, with `{{var}}` interpolation
- `useT()` hook: used in Dashboard, Meals, Workouts, Calendar, Progress, UserProfile, Billing, Onboarding, ShoppingList
- Language toggle persisted in `localStorage` (`goaliq_lang`)
- `useTrialCopy()` in `src/lib/i18n.ts`: English-only Stripe trial copy (intentionally separate)
- Vite Fast Refresh warning for `useLanguage` export is non-blocking

### PWA
- Web App Manifest at `/public/manifest.json`
- Installable on mobile

## API Routes

All routes prefixed with `/api`:

- `GET /healthz` - Health check
- `GET /auth/user` - Current auth state
- `GET /login` - Replit OIDC login
- `GET /callback` - OIDC callback
- `GET /logout` - Logout
- `GET/POST /onboarding` - User onboarding profile
- `GET/POST /meals` - Get / generate meal plan
- `POST /meals/replace-ingredient` - Replace ingredient in meal
- `GET/POST /workouts` - Get / generate workout plan
- `GET /calendar` - Get calendar events (by month)
- `POST /calendar/complete` - Toggle workout completion
- `GET/POST /progress` - Get / update progress (weight)
- `GET /plans` - List Stripe subscription plans
- `GET /subscription` - Current user's subscription tier (requires auth)
- `POST /checkout` - Create Stripe checkout session
- `POST /portal` - Create Stripe billing portal session
- `POST /stripe/webhook` - Stripe webhook receiver (raw body)

## Database Tables

- `users` - Auth users
- `sessions` - Auth sessions
- `onboarding_profiles` - User health + goal profiles
- `meal_plans` - Generated meal plans (JSON)
- `workout_plans` - Generated workout plans (JSON)
- `calendar_events` - Workout calendar events
- `weight_entries` - Weight logging history

## QA Infrastructure

### Phase 1 — Static QA (fast, ~5s)
- **CLI script**: `node scripts/qa-test.js` (or `pnpm run qa`) — checks env vars, local PG tables, Supabase tables, API auth guards, and frontend routes. Prints color-coded pass/fail report.
- **Server endpoint**: `GET /api/qa` — server-side checks returning `{ timestamp, summary, results[] }`.
- **In-app button**: "🔍 QA" in sidebar/mobile header (dev only, `.replit.dev` domains).

### Phase 2 — Full E2E Test Suite (15 tests, ~1-2 min)
- **CLI script**: `node scripts/e2e-test.js` (or `pnpm run e2e`) — signs in as the test user and calls the server-side E2E endpoint.
- **Server endpoint**: `GET /api/qa/e2e` — reads `Authorization: Bearer <token>` header, uses the caller's session to run 15 end-to-end tests covering: auth, profile, food prefs, AI meal plan generation, meal DB save, AI workout generation, workout DB save, shopping list validation, calendar complete, flex days, progress logging, streak calculation, Stripe checkout URL, language (Spanish) check, and data cleanup.
- **In-app button**: "🤖 Test E2E completo" inside the QA modal (below the Phase 1 results). Calls `/api/qa/e2e` with the current session token. Shows a loading spinner during the ~1-2 minute run, then displays the full report inline.
- **Auth strategy**: The E2E endpoint uses the caller's Supabase JWT (no separate test user needed). It falls back to `test-qa@goaliq.app` / `TestQA2026!` if no header is supplied.
- **Cleanup**: TEST 15 only removes data created during the test run (meal plan by week_start, workout plan by week_start, recent calendar events, recent weight entry, flex days). User profile and food preferences are preserved.

## Environment Variables

- `DATABASE_URL` - PostgreSQL connection string
- `AI_INTEGRATIONS_OPENAI_BASE_URL` - OpenAI API base URL (via Replit AI Integrations)
- `AI_INTEGRATIONS_OPENAI_API_KEY` - OpenAI API key (via Replit AI Integrations)
- `REPL_ID` - Replit REPL ID (used for OIDC client_id)
- `REPLIT_DB_URL` - Used for session storage (KV store)
